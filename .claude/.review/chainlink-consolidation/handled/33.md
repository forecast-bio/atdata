# Issue #33: Implement record ingestion from ATProto firehose

## REVIEW

See elsewhere.

## Verdict: ARCHIVE

## Evidence
Firehose subscription was never implemented because direct PDS access proved
sufficient. The `Atmosphere` client communicates directly with PDS endpoints for
CRUD operations on schema, dataset, and lens records. The `_pds_endpoint_cache`
in `atmosphere/__init__.py` optimizes repeated DID-to-PDS resolution. Record
listing and retrieval use standard ATProto `com.atproto.repo.listRecords` and
`com.atproto.repo.getRecord` calls. There is no need for a firehose subscriber
given the current use case of explicit publish/discover/load workflows.
