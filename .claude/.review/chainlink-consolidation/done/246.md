# Issue #246: Add Dataset.from_index_entry() static constructor for polymorphic loading

## Verdict: ARCHIVE

## Evidence
This issue proposed a `Dataset.from_index_entry()` or `Dataset.from_index()` static constructor for polymorphic dataset loading from IndexEntry objects. The use case is fully superseded by `load_dataset()` in `_hf_api.py`, which supports both `@handle/name` indexed path resolution and `at://` AT Protocol URI resolution. The `_resolve_indexed_path()` function handles the full workflow: parsing `@handle/dataset[@version]` notation, looking up entries via `index.get_label()` or `index.get_dataset()`, resolving S3 credentials from the data store, and transforming URLs via `store.read_url()`. The `_resolve_at_uri()` function handles AT URIs by fetching the record, determining storage type (blobs, HTTP, S3, external), resolving shard URLs, and optionally decoding the schema from the referenced schema record. Both paths produce a fully configured `Dataset` instance without requiring a separate static constructor.
