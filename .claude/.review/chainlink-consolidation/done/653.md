# Issue #653: Implement content checksums for all storage backends

**Priority:** medium
**Status:** open -> CLOSED
**Created:** 2026-02-04

## History

Tracked as GitHub issue #16. Covers content checksum implementation for LocalDiskStore, S3DataStore, and PDSBlobStore to ensure data integrity.

## Current State Assessment

Content checksums are fully implemented across all storage backends:

- `src/atdata/stores/_disk.py` -- LocalDiskStore checksum support
- `src/atdata/stores/_s3.py` -- S3DataStore checksum support
- `src/atdata/atmosphere/store.py` -- PDSBlobStore checksum support
- `src/atdata/repository.py` -- `_AtmosphereBackend.insert_dataset()` accepts and forwards checksums
- `src/atdata/index/_index.py` -- Index-level checksum integration
- `src/atdata/_helpers.py` -- Checksum verification utilities
- `src/atdata/cli/__init__.py` -- `atdata verify` CLI command for checksum verification
- `src/atdata/lexicons/__init__.py` -- Checksum types in lexicon definitions

PR #29 specifically fixed the gap in `publish_with_blob_refs()` where checksums were not being stored.

## Decision

**CLOSE** -- Content checksums are implemented end-to-end across all backends with CLI verification support.
