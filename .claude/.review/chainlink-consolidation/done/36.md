# Issue #36: Design code generation template system

## Verdict: ARCHIVE

## Evidence
The original concept of a template-based code generation system has been
superseded by two complementary approaches: (1) the `lexicons/` package which
provides canonical Lexicon JSON definitions as dataclass-like structures loaded
at runtime, and (2) `_schema_codec.py` which dynamically generates Python types
from stored schemas without a template engine. The lexicons package defines all
ATProto record types (`science.alt.dataset.schema`, `.record`, `.lens`, etc.)
and provides `load_lexicon()` / `all_lexicon_ids()` APIs. Dynamic type
generation in `_schema_codec.py` uses `schema_to_type()` which builds
`PackableSample` subclasses programmatically. No Jinja or template engine was
needed.
