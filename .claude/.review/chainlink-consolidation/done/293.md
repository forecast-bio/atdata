# Issue #293: S3 URI scheme issues: https vs s3 for wds streaming + host details

## Verdict: CLOSE

## Evidence
`S3DataStore.read_url()` in `stores/_s3.py` converts `s3://` URIs to `https://` URLs using the `AWS_ENDPOINT` from credentials. When a custom endpoint is configured (MinIO, Cloudflare R2, etc.), it strips the `s3://` prefix and prepends the endpoint URL, producing a streaming-ready HTTPS URL that WebDataset can consume directly. For standard AWS S3 (no custom endpoint), URLs are returned unchanged since WebDataset's built-in s3fs handles them natively. The fix is verified by live integration tests in `test_s3_live.py` with `TestURLResolution` that asserts the converted URL format matches `{endpoint}/{bucket}/{path}`.
