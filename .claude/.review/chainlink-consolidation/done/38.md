# Issue #38: Add CLI for code generation

## Verdict: ARCHIVE

## Evidence
A dedicated CLI command for code generation was not implemented. Instead, code
generation is handled at the API level via `_schema_codec.py` (runtime type
generation) and `_stub_manager.py` (IDE stub management). The CLI does include
schema-related commands -- `schema show` displays a dataset's schema and
`schema diff` compares two schemas -- but these are inspection tools rather than
code generators. The API-level approach was chosen because generated types are
created dynamically at runtime when loading atmosphere datasets, making a
separate CLI codegen step unnecessary.
