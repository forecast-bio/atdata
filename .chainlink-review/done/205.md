# Issue #205: Add cleanup utilities for test records

## Verdict: CLOSE

## Evidence
Cleanup utilities are implemented at multiple levels. The `_cleanup_records()` helper function in `test_atproto_live.py` iterates over all records in a collection and deletes those matching the RUN_ID prefix (best-effort, skipping failures). `TestZZZCleanup` class runs last by alphabetical naming convention and sweeps both schema and dataset collections for any leftover test records. Individual tests also perform inline cleanup by calling `delete_record()` after assertions. For infrastructure services, the `conftest.py` fixtures provide teardown logic: MinIO bucket cleanup (paginated object deletion + bucket deletion), PostgreSQL table truncation with `RESTART IDENTITY CASCADE`, and Redis database flush via `flushdb()`.
