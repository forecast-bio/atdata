# Issue #34: Implement search and query API

## REVIEW

See elsewhere.

## Verdict: ARCHIVE

## Evidence
The original concept of a search/query API has been superseded by the manifest
and query system implemented in the `manifest/` module, combined with record
discovery methods like `list_all()` and `find_by_schemas()`. The manifest system
provides per-shard metadata queries via `ManifestField` descriptors and a
`QueryExecutor` that supports filtering, aggregation, and selection. Record
discovery on the atmosphere side uses `list_all()` and `find_by_schemas()` for
schema-based lookup. This approach integrates directly into the data pipeline
rather than requiring a separate search service.
